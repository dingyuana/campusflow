🎓 教学大纲  
《基于 Supabase + Vercel + uv + Git 的XX实训系统全流程开发大纲》

课程概述
本课程通过构建一个XX实训系统，让学生掌握现代软件工程实践，包括但不限于：
- 集成 Supabase 数据库进行数据持久化与实时更新。
- 前端部署于 Vercel 平台。
- 使用 uv 管理 Python 虚拟环境及依赖。
- 遵循 Git 工作流规范进行版本控制与团队协作。

目标受众
面向高校计算机、人工智能或软件工程专业高年级本科生/研究生。

课程周期
建议为期 8 周（每周 10 小时）或 2 周集中实训（80 小时）

技术栈
- 后端：FastAPI (Python)  + Supabase + uv
- 前端：Next.js + Tailwind CSS + Vercel AI SDK
- 数据库：Supabase (PostgreSQL + Auth + Realtime)
- 版本控制：Git + GitHub

一、课程结构

▶ 阶段 1：项目启动与环境搭建（第 1 周）
- 目标：理解项目背景，完成本地开发环境配置
- 内容：
  - 项目介绍与目标设定
  - 安装与配置 uv，创建虚拟环境
  - 克隆项目仓库并初始化 .env 文件
  - 设置 Supabase 项目与获取 API Key
  - 创建 GitHub 仓库，配置基础 Git 流程

▶ 阶段 2：需求分析与用户故事定义（第 2 周）
- 目标：明确系统功能需求
- 内容：
  - 分析用户角色及其需求
  - 定义用户故事与功能清单
  - 初步设计系统架构图

▶ 阶段 3：后端开发基础（第 3 周）
- 目标：搭建 FastAPI 后端框架，实现基本功能
- 内容：
  - 快速入门 FastAPI
  - 实现用户认证模块（集成 Supabase Auth）
  - 设计并实现 RESTful API 接口
  - 添加单元测试案例

▶ 阶段 4：数据库设计与实现（第 4 周）
- 目标：在 Supabase 中设计并实现所需数据库表结构
- 内容：
  - 设计数据库表结构（如 profiles, conversations 等）
  - 使用 SQL 在 Supabase 中创建表，并设置行级安全策略
  - 编写 CRUD 操作函数并与后端 API 对接
  - 实现数据库操作的单元测试

▶ 阶段 5：前端开发基础（第 5 周）
- 目标：构建 Next.js 前端应用框架
- 内容：
  - Next.js 基础教程
  - 实现用户登录页面
  - 连接后端 API 获取数据并在前端展示
  - 添加样式与交互效果

▶ 阶段 6：用户真实需求实施

▶ 阶段 7：测试策略与质量保证（第 7 周）
- 目标：确保系统的稳定性和安全性
- 内容：
  - 单元测试、集成测试、E2E 测试
  - 使用 OWASP ZAP 进行安全扫描
  - 性能测试与优化

▶ 阶段 8：部署与发布（第 8 周）
- 目标：将项目部署上线
- 内容：
  - 配置 CI/CD 流程
  - 部署后端至 Render 或 Fly.io
  - 前端部署至 Vercel
  - 最终检查与演示准备

二、评估标准

项目成果
- 功能完整性（40%）
- 架构合理性（30%）
- 安全与鲁棒性（20%）
- 文档与演示（10%）

个人贡献
- 团队合作能力
- 解决问题的能力
- 学习态度与积极性
